-pull request (PR)

通知：告知相關人士一起檢查
討論：使討論不分散，集中在某個畫面上
修改：進行錯誤或是改善，根據討論而做出的修改 (follow-up commit) 標示

-pull request 流程
1. 開發者在自己的分支上開發一個功能&完成
2. 開發者將自己的分支提交上 (git push origin XXX)
3. 開發者建立 PR ，告知大家來做檢查
4. 若同意修改，可將分支合併回master分支
ps. 可以Fork一份原作的專案到自己的 GitHub 帳號底下修改，
修改完後可pull request 到原作者
